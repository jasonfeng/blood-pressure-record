## 1. 项目概述

### 1.1 目标

开发一款 Android 血压记录应用，帮助用户便捷地记录每日早晚血压值，支持历史查看和飞书云端同步。

### 1.2 核心功能

1. 每天早晚记录一次血压值，系统自动判断时段
2. 数据格式：高压/低压（例：120/80）
3. 支持按时间段查看血压记录（今日、近一周、近一个月等）
4. 本地存储 + 飞书实时同步
5. 图表 + 列表双视图展示
6. Material Design 3 简洁现代风格 + 深色模式支持

### 1.3 目标用户

- 自己使用
- 对智能手机非常熟悉

## 3. 数据模型

### 3.1 血压记录实体

```kotlin
@Entity(tableName = "blood_pressure_records")
data class BloodPressureRecord(
    @PrimaryKey(autoGenerate = true)
    val id: Long = 0,
    
    val date: LocalDate,           // 日期
    val period: Period,            // 时段 (MORNING/EVENING)
    val systolic: Int,             // 高压 (收缩压)
    val diastolic: Int,            // 低压 (舒张压)
    val heartRate: Int?,           // 心率 (可选)
    val note: String?,             // 备注 (可选)
    val syncStatus: SyncStatus,    // 同步状态
    val syncTime: LocalDateTime?,  // 飞书同步时间
    val createdAt: LocalDateTime,  // 创建时间
    val updatedAt: LocalDateTime   // 更新时间
)

enum class Period { MORNING, EVENING }
enum class SyncStatus { PENDING, SYNCED, FAILED }
```

### 3.2 时段自动识别逻辑

```kotlin
fun determinePeriod(currentTime: LocalTime): Period {
    return when (currentTime.hour) {
        in 4..11 -> Period.MORNING   // 早上 4:00-11:00
        in 17..22 -> Period.EVENING  // 晚上 17:00-22:00
        else -> {
            // 非标准时段，智能判断
            if (currentTime.hour < 12) Period.MORNING 
            else Period.EVENING
        }
    }
}
```

## 5. 飞书同步

### 5.1 同步策略

采用 **本地优先 + 实时同步** 策略：

```
用户操作 (记录血压)
       ↓
   本地存储 (Room)
       ↓
   实时同步 → 飞书多维表格
       ↓
   同步状态反馈 (成功/失败/重试中)
```

### 5.2 飞书数据格式

使用 **飞书多维表格** 存储：

| 日期 | 时段 | 高压 | 低压 | 心率 | 备注 | 同步时间 |
|------|------|------|------|------|------|----------|
| 2024-04-02 | 早上 | 120 | 80 | 72 | 饭后 | 2024-04-02 07:30:00 |
| 2024-04-02 | 晚上 | 118 | 78 | 70 | - | 2024-04-02 22:15:00 |

### 5.3 同步机制

| 场景 | 处理方式 |
|------|----------|
| 记录保存 | 立即发起同步请求 |
| 网络不可用 | 本地缓存，网络恢复后重试 |
| 飞书 API 失败 | 指数退避重试 (1s, 2s, 4s, 8s...) |
| 数据冲突 | 以本地时间最新为准 |

### 5.4 技术实现

- 使用飞书开放平台 API
- 需要用户授权 OAuth 2.0
- 首次使用引导用户授权
- 存储 Access Token (需要刷新)

## 6. 错误处理

| 错误类型 | 用户反馈 | 恢复策略 |
|----------|----------|----------|
| 网络不可用 | "网络不可用，数据已本地保存" | 缓存队列，恢复后重试 |
| 飞书授权失效 | "飞书连接已过期，请重新授权" | 跳转授权页面 |
| API 请求失败 | "同步失败，将自动重试" | 指数退避 (最多3次) |
| 数据校验失败 | "血压数值超出合理范围" | 提示正确范围 |
| 数据库写入失败 | "保存失败，请重试" | 显示错误详情 |
